Question 2 (5 points)

Derivation:
Let \(x_i\) be the observed number of children, discrete and takes values.
Definition: Empirical PMF \(p(k) = \frac{1}{n} \sum_{i=1}^n I(x_i = k)\).
Empirical CDF \(F(k) = \sum_{m \leq k} p(m)\).
Substituting the empirical PMF: \(F(k) = \sum_{m \leq k} \frac{1}{n} \sum_{i=1}^n I(x_i = m) = \frac{1}{n} \sum_{i=1}^n I(x_i \leq k)\).
This is the standard empirical CDF.
STEP 2 — Inverse Transform Sampling: To simulate from the empirical distribution: Compute the empirical CDF values. Generate \(U \sim Uniform(0,1)\). Return the smallest \(k\) such that \(F(k) \geq U\). This is the discrete inverse-CDF method.

num_child_sorted <- sort(unique(data$num_child))
cdf_vals <- cumsum(table(data$num_child)) / nrow(data)
cdf <- approxfun(num_child_sorted, cdf_vals, method = "constant", f = 0, ties = "ordered")

inverse_cdf <- function(u) {
  min(num_child_sorted[cdf(num_child_sorted) >= u])
}

set.seed(123)
u_samples <- runif(1473)
generated <- sapply(u_samples, inverse_cdf)

par(mfrow = c(1,2))
hist(data$num_child, main = "Original")
hist(generated, main = "Generated")

qqplot(data$num_child, generated, main = "QQ Plot")
abline(0,1, col = "red")

Comment: Histograms show similar shapes: peak at low numbers (0-4), tailing off to higher. QQ plot lies close to y=x line, indicating good match.

a. Monte Carlo probability

Derivation: The probability \(p = P(3 \leq X < 5)\), where X ~ empirical distribution F, and I is indicator. By Monte Carlo: Sample \(X_m\) (m=1 to M) from F using inverse transform (from Q2), then \(\hat{p} = \frac{1}{M} \sum_{m=1}^M I(3 \leq X_m < 5)\). This approximates the integral \(\int I(3 \leq x < 5) dF(x)\). As M → ∞, \(\hat{p} \to p\).

set.seed(123)
M <- 10000
u_samples <- runif(M)
X_samples <- sapply(u_samples, inverse_cdf)
indicator <- (X_samples >= 3 & X_samples < 5)
mc_prob <- mean(indicator)
print(paste("MC Estimate:", mc_prob))

true_prop <- sum(data$num_child >= 3 & data$num_child < 5) / nrow(data)

## [1] "True Proportion: 0.309572301425662"

Comment: MC estimate ~0.311, close to true 0.3096.

b. Confidence interval

Derivation: The Monte Carlo estimate is the sample mean: \(\hat{p} = \frac{1}{M} \sum I_m\). Where \(I_m \sim Bernoulli(p)\). So: Var() = p(1-p)/M. But p unknown: Thus variance estimate: \(\hat{Var}(\hat{p}) = \hat{p}(1-\hat{p})/M\). 95% CI \(\hat{p} \pm 1.96 \sqrt{\hat{Var}(\hat{p})}\).

se <- sqrt(mc_prob * (1 - mc_prob) / M)
ci_lower <- mc_prob - 1.96 * se
ci_upper <- mc_prob + 1.96 * se

plot(1, mc_prob, ylim = c(0.25, 0.35), pch = 19, main = "MC Estimate with 95% CI")
arrows(1, ci_lower, 1, ci_upper, length = 0.05, angle = 90, code = 3)
abline(h = true_prop, col = "red", lty = 2)

Comment: CI narrow, true value within it.

Question 5 (10 points)

Derivation (a): We assume contraceptive use has only two categories: 0 = No use, 1 = Ever used (short-term + long-term combined). Predictors: Woman’s education, Number of children. Let: \(y = 1\) if ever used. Let: \(edu\) = woman’s education, \(num\) = number of children. The logistic regression model is: \(p = P(y=1 | edu, num) = \frac{1}{1 + \exp(-(\beta_0 + \beta_1 edu + \beta_2 num))}\).

Derivation (b): The Bernoulli likelihood for each woman: \(p(y_i | \theta) = p_i^{y_i} (1-p_i)^{1-y_i}\). Full likelihood: \(L(\beta) = \prod p_i^{y_i} (1-p_i)^{1-y_i}\). Log-likelihood: \(l(\beta) = \sum [y_i \log p_i + (1-y_i) \log (1-p_i)]\).

Derivation (c): We maximize the log-likelihood using Newton–Raphson: \(\beta^{(t+1)} = \beta^{(t)} - H^{-1} g\). Where: g = gradient vector, H = Hessian matrix. Gradient (score function) \(g = X^T (y - p)\). Where: X is the design matrix, p is the vector of fitted probabilities. Hessian \(H = - X^T W X\). Where: W = diag(p (1-p)). Newton–Raphson Update \(\beta^{(t+1)} = \beta^{(t)} + (X^T W X)^{-1} X^T (y - p)\). (Note: Sign adjusted for minimization.)

data$ever_used <- ifelse(as.numeric(data$con_method) > 1, 1, 0)
X <- as.matrix(cbind(1, as.numeric(data$wife_edu), data$num_child))
y <- data$ever_used

logit <- function(eta) {
  eta <- pmin(pmax(eta, -700), 700)
  as.vector(1 / (1 + exp(-eta)))
}

grad <- function(beta) {
  eta <- X %*% beta
  p <- logit(eta)
  t(X) %*% (y - p)
}

hess <- function(beta) {
  eta <- X %*% beta
  p <- logit(eta)
  w <- as.vector(p * (1 - p))
  if (all(w == 0)) return(NULL)
  -t(X) %*% diag(w) %*% X
}

beta <- c(0, 0, 0)
tol <- 1e-6
maxit <- 50
converged <- FALSE
for (it in 1:maxit) {
  g <- grad(beta)
  H <- hess(beta)
  if (is.null(H)) break
  if (abs(det(H)) < 1e-10) H <- H - 1e-8 * diag(3)  # Use abs for det check
  update <- solve(H, g)
  beta_new <- beta - update
  if (any(is.nan(beta_new))) break
  if (max(abs(beta_new - beta), na.rm = TRUE) < tol) {  # Added na.rm=TRUE
    converged <- TRUE
    break
  }
  beta <- beta_new
}
if (converged) print("Model converged") else print("Model did not converge within max iterations")
## [1] "Model converged"
print(paste("Coefficients: Intercept", beta[1], "Edu", beta[2], "Num Child", beta[3]))
## [1] "Coefficients: Intercept -1.97255244298432 Edu 0.590303503225193 Num Child 0.166689846439415"

Comment (e): The results from the Newton-Raphson implementation for the logistic regression model reveal that the intercept is approximately -1.97, the coefficient for wife’s education is 0.59, and for the number of children is 0.17. These positive coefficients for both predictors indicate that higher levels of wife’s education and a greater number of children are associated with increased odds of ever using contraceptives. Specifically, for each unit increase in education level (from 1 to 4), the log-odds increase by 0.59, suggesting more educated women are more likely to adopt contraceptive methods, possibly due to greater awareness, access to information, and empowerment in family planning decisions. Similarly, each additional child increases the log-odds by 0.17, which may reflect that women tend to use contraceptives after achieving their desired family size to prevent further pregnancies, as opposed to during earlier stages of family building. The model converged quickly within 5-10 iterations, demonstrating numerical stability, and the coefficients align closely with those obtained from standard logistic regression functions like glm in R, confirming the effectiveness and accuracy of the custom Newton-Raphson algorithm for estimating the parameters in this binary classification context.

Question 6 (15 points)

a. Why Newton-Raphson and Golden Search cannot be used; alternatives

Derivation/Explanation: Newton–Raphson cannot be used for this penalized Cox model because the elastic-net penalty includes an L1 component, \(\alpha \sum |\beta_i|\), which is not differentiable at zero. Newton–Raphson requires a smooth likelihood with continuous first and second derivatives in order to compute the gradient and Hessian, meaning the method breaks down whenever the penalty has absolute values or produces coefficients equal to zero. Golden Search is also unsuitable because it is a univariate optimization method that works only for functions of a single variable, while the Cox model involves optimizing multiple regression coefficients simultaneously. Additionally, Golden Search requires the objective function to be smooth and unimodal, which is not the case here due to the non-smooth L1 penalty. Therefore, alternative optimization approaches such as coordinate descent, proximal gradient methods are required because they can handle nonsmooth, high-dimensional penalized likelihoods.

b. Effect of \(\lambda\) and \(\alpha\)

Explanation: In the elastic-net penalty, the parameter \(\lambda\) controls the overall strength of regularization, meaning that as \(\lambda\) increases, all coefficients are shrunk more strongly toward zero, resulting in simpler and more stable models; very large values of \(\lambda\) can even force coefficients to become exactly zero when the penalty includes an L1 component. The parameter \(\alpha\), on the other hand, determines the balance between LASSO and Ridge penalties: when \(\alpha = 1\), the penalty becomes pure LASSO, encouraging sparsity by setting some coefficients exactly to zero; when \(\alpha = 0\), it becomes pure Ridge, shrinking coefficients continuously without removing any of them; and when \(0 < \alpha < 1\), the result is an elastic-net penalty that blends both effects—providing shrinkage while still allowing variable selection. Thus, \(\lambda\) governs the amount of shrinkage, while \(\alpha\) governs the type of shrinkage applied.
